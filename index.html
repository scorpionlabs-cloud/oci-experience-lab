Suggested Starter Architecture

This lab assumes a simple, production-inspired layout in OCI:

  • 1 VCN with public + private subnets
  • Internet Gateway for load balancers only
  • NAT Gateway for outbound access from private subnets
  • Compute instances and OKE nodes in private subnets
  • Object Storage for artifacts and backups
  • Autonomous Database or MySQL HeatWave for data
  • Compartments to separate environments (dev / test / prod)
  • Optional: private interconnect (FastConnect / VPN) to another cloud or on-prem network

This page is a learning aid; validate quotas, IAM policies, and security with your cloud team before deploying.

Getting Started With the Lab

  1. Create or use an existing OCI tenancy and user with limited test permissions.
  2. Install the OCI CLI and configure a profile with API keys.
  3. Clone this GitHub repository to view all snippets and Terraform examples.
  4. Open each lab here, copy the sample commands, and run them from your terminal or pipeline.
  5. Clean up resources when finished to avoid unnecessary cost.

Tip: start in a non-production compartment with strict budgets and alerts.
